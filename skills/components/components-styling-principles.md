# Components: Styling Principles

> **Context**: CSS architecture avoiding conflicts and hardcode.
> **SSOT**: `styles/`

## 1. Core Rules
1.  **Bootstrap First**: Use utility classes (`d-flex`, `mb-3`) before custom CSS.
2.  **Neutral Tones**: Use `text-secondary` + `opacity` for adaptation to Light/Dark themes. Avoid HEX/RGB hardcoding.
3.  **Backgrounds**: Use `bg-body`, `bg-surface`, `bg-transparent`.
4.  **Borders**: `border-secondary` (subtle).

## 2. Scoping
- **Prefix**: `avto-{hash}` generated by `hash-generator.js`.
- **Usage**:
  ```css
  /* Good */
  .avto-Xy7z .custom-element { ... }
  ```

## 3. Workflow
1.  Apply Bootstrap utilities in template.
2.  If insufficient, add `classesAdd` prop.
3.  If custom CSS needed, document values in `styles/wrappers/`.

## 4. File Map
- `@styles/layout/`: Global layout.
- `@styles/wrappers/`: Component-specific overrides.
