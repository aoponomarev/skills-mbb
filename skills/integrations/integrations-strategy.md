# integrations-strategy

> Источник: `docs/doc-external-integrations.md`

## Принципы интеграции

- Интеграция по мере необходимости, не "про запас".
- Отказоустойчивость по умолчанию (primary → secondary → tertiary → local).
- Гео‑оптимизация: РФ/СНГ → Yandex Cloud, глобально → Cloudflare.
- Единый источник правды в `core/config/` и при необходимости `core/config/integration-config.js`.
- Проверка актуальной документации провайдеров перед проектированием.

## Архитектура отказоустойчивости

Централизованный узел интеграции (планируется):
- `core/api/integration-manager.js`
- мониторинг доступности
- автоматическое переключение
- кэширование и метрики

## Команда "EI:"

Триггер для анализа интеграций: сравнение провайдеров, проектирование фолбэка, план реализации. Реализация — только по явной команде пользователя.
