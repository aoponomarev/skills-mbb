# Overview

This skill focuses on setting up and managing automated infrastructure monitoring systems to ensure the reliability and performance of cloud-based and on-premise infrastructure.

## Context

Infrastructure monitoring is crucial for maintaining the health and performance of IT systems. Automated monitoring helps in proactive issue detection, reducing downtime, and improving system reliability. This skill is designed to integrate seamlessly with existing infrastructure and monitoring tools.

## Guidelines

1. **Setup Monitoring Tools**: Install and configure monitoring tools such as Prometheus, Grafana, or Nagios.
2. **Define Metrics**: Identify key performance indicators (KPIs) and metrics to monitor, such as CPU usage, memory usage, disk I/O, and network traffic.
3. **Configure Alerts**: Set up alerts for critical metrics to notify the team of any anomalies or potential issues.
4. **Integrate with Existing Systems**: Ensure the monitoring tools are compatible with the current infrastructure and can gather data from all relevant sources.
5. **Automate Reports**: Generate regular reports on system performance and health to share with stakeholders.

## Examples

- **Example 1**: Setting up Prometheus to monitor CPU and memory usage on a cloud-based infrastructure.
- **Example 2**: Configuring Grafana dashboards to visualize network traffic and disk I/O metrics.
- **Example 3**: Creating alerts in Nagios for critical system metrics and integrating them with the team's communication tools.
